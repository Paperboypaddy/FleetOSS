# Database Schema

FleetOSS uses **PostgreSQL 16** with the **PostGIS 3.4** extension for geospatial queries.

## Design Decisions

- **snake_case** column names in the database (`.js` files use `.js` extension for ESM)
- **camelCase** in TypeScript via Drizzle ORM field mapping
- **UUID primary keys** for all tables (generated via `uuid_generate_v4()`)
- **JSONB** for flexible attributes on most entities
- **TIMESTAMPTZ** (timestamp with timezone) for all date fields
- **GiST index** on position geometry for fast spatial queries

## Tables

### devices
Stores registered GPS devices/trackers.

| Column | Type | Notes |
|--------|------|-------|
| id | UUID | PK, auto-generated |
| name | TEXT | Human-readable device name |
| unique_id | TEXT | UNIQUE. Device serial/IMEI/app identifier |
| plate | TEXT? | License plate (vehicles) |
| vin | TEXT? | Vehicle identification number |
| status | TEXT | `'online'`, `'offline'`, or `'unknown'` |
| attributes | JSONB | Flexible device metadata |
| created_at | TIMESTAMPTZ | Auto-set |
| updated_at | TIMESTAMPTZ | Auto-set |

### positions
Raw GPS position records. This is the fastest-growing table — expect millions of rows.

| Column | Type | Notes |
|--------|------|-------|
| id | UUID | PK |
| device_id | UUID | FK → devices.id |
| latitude | DOUBLE | Required. WGS84 decimal degrees |
| longitude | DOUBLE | Required. WGS84 decimal degrees |
| altitude | DOUBLE? | Meters above sea level |
| speed | DOUBLE? | mph (or km/h depending on device) |
| bearing | DOUBLE? | Degrees, 0–360 |
| accuracy | DOUBLE? | GPS accuracy in meters |
| course | DOUBLE? | Direction of travel |
| ignition | BOOLEAN? | Engine on/off |
| engine_hours | DOUBLE? | Total engine run time |
| odometer | DOUBLE? | Total vehicle distance |
| fuel_level | DOUBLE? | Percentage 0–100 |
| battery_level | DOUBLE? | Device battery percentage |
| rpm | DOUBLE? | Engine RPM (OBD-II vehicles) |
| fuel_consumption | DOUBLE? | Instantaneous consumption |
| valid | BOOLEAN | Default true. False for invalid GPS fixes |
| protocol | TEXT | Protocol identifier (e.g. 'http-json', 'nmea') |
| attributes | JSONB | Protocol-specific data |
| device_timestamp | TIMESTAMPTZ | Time reported by device |
| server_timestamp | TIMESTAMPTZ | Time received by server |

**Indexes:**
- `idx_positions_device_time` — B-tree on `(device_id, device_timestamp DESC)` for time-range queries per device
- `idx_positions_geom` — GiST on `ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)` for spatial queries

### trips
Auto-detected trips from GPS position sequences.

| Column | Type | Notes |
|--------|------|-------|
| id | UUID | PK |
| device_id | UUID | FK → devices.id |
| start_position_id | UUID? | FK → positions.id |
| end_position_id | UUID? | FK → positions.id |
| start_time | TIMESTAMPTZ | Trip start time |
| end_time | TIMESTAMPTZ | Trip end time |
| start_lat | DOUBLE | Starting latitude |
| start_lng | DOUBLE | Starting longitude |
| end_lat | DOUBLE | Ending latitude |
| end_lng | DOUBLE | Ending longitude |
| start_address | TEXT? | Reverse-geocoded start address |
| end_address | TEXT? | Reverse-geocoded end address |
| distance | DOUBLE | Miles traveled |
| duration | INTEGER | Duration in seconds |
| avg_speed | DOUBLE | Average speed (mph) |
| max_speed | DOUBLE | Maximum speed (mph) |
| stop_duration | INTEGER? | Idle time at end (seconds) |
| attributes | JSONB | Additional trip data |

### geofences
Geographic boundaries for zone-based alerts.

| Column | Type | Notes |
|--------|------|-------|
| id | UUID | PK |
| name | TEXT | Human-readable name |
| type | TEXT | `'circle'`, `'polygon'`, or `'polyline'` |
| latitude | DOUBLE? | Center latitude (circle) |
| longitude | DOUBLE? | Center longitude (circle) |
| radius | DOUBLE? | Radius in meters (circle) |
| polygon | JSONB? | Array of [lat, lng] coordinates (polygon) |
| polyline | JSONB? | Array of [lat, lng] coordinates (polyline) |
| attributes | JSONB | Additional geofence metadata |

### events
System events generated by device activity.

| Column | Type | Notes |
|--------|------|-------|
| id | UUID | PK |
| device_id | UUID | FK → devices.id |
| type | TEXT | Event type string |
| position_id | UUID? | FK → positions.id. Related position |
| geofence_id | UUID? | Related geofence |
| attributes | JSONB | Event-specific data |
| time | TIMESTAMPTZ | When the event occurred |

## PostGIS Setup

The PostGIS extension is created by the migration script:
```sql
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

The spatial index is created on positions:
```sql
CREATE INDEX IF NOT EXISTS idx_positions_geom
ON positions USING GIST
(ST_SetSRID(ST_MakePoint(longitude, latitude), 4326));
```

This enables queries like:
```sql
-- Find all positions within 1km of a point
SELECT * FROM positions
WHERE ST_DWithin(
  ST_SetSRID(ST_MakePoint(longitude, latitude), 4326),
  ST_SetSRID(ST_MakePoint(-116.945, 47.718), 4326),
  1000
);

-- Find the closest device to a point
SELECT device_id, ST_Distance(
  ST_SetSRID(ST_MakePoint(longitude, latitude), 4326),
  ST_SetSRID(ST_MakePoint(-116.945, 47.718), 4326)
) as dist
FROM positions
ORDER BY dist
LIMIT 1;
```
